Site Setup Guide

How to Organize Your Juice & Smoothie Recipe Site

1. Use Pages for Each Recipe, Not Posts

Why?

  • Recipes are typically evergreen, standalone content.
  • Pages are great for fixed, organized content like recipes — easily linked and structured.
  • Posts are more suited for blog-style, time-sensitive updates.

How?
Create each recipe as a separate Page with a clean URL like /recipes/peppermint-blaze/ or /juices/peppermint-blaze/.

2. Create a “Recipes” Parent Page / Folder

Structure your site hierarchy so that all recipes live under a top-level “Recipes” or “Juices & Smoothies” page. This keeps everything tidy, e.g. /recipes/, /recipes/peppermint-blaze/.

Publii lets you nest pages, so you can have:

  • /recipes/ (main landing page listing all recipes, maybe with thumbnails and quick links)
  • /recipes/peppermint-blaze/ (individual recipe page)

3. Landing Page with Recipe Index

Create a landing page (e.g. your /recipes/ page) that lists all recipes in a visually appealing grid or list with images and short intros. You can add search or filtering later (Publii supports basic search functionality).

4. Metadata & SEO

  • Use Publii’s SEO options per page: unique title tags & meta descriptions.
  • Embed Schema.org Recipe markup — which you’re already generating — directly in each recipe page’s source.
  • Include descriptive alt text on all images for accessibility and SEO.

5. Use Categories or Tags (Optional)

Publii supports tags and categories, but for static recipe sites without a backend, they can be tricky. You can:

  • Use tags for ingredients or dietary needs (e.g., “high-antioxidant,” “post-workout”).
  • Or keep it simple with clear URLs and a well-organized landing page.

6. Media Folder Organization

Store images for each recipe in dedicated folders like /media/recipes/peppermint-blaze/ to keep your repository clean. Publii automates this if you upload images via its media manager per page.

7. Performance

  • Publii generates static HTML—fast loading, no server hassle.
  • Compress images for quicker loading.
  • Keep CSS minimal and efficient.

8. Workflow Tip

After creating or editing recipes in Publii, preview locally to check layout, schema markup, and styling. Once happy, push to GitHub Pages with one click.
